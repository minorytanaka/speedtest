# Speedtest on CLI

Скрипт-замерятель скорости интернета. Делает 10 последовательных запросов к
заданному URL (тяжёлая картинка/файл), считает среднее время запроса, объём
скачанных данных и итоговую скорость в МБ/с.

## Как работает

Вся логика сводится к формуле **скорость = объём данных / время**:

1. Засекаем монотонный таймер (`time.perf_counter`), делаем GET-запрос и
   дожидаемся, пока скачается всё тело ответа.
2. Останавливаем таймер, получаем длительность запроса и размер скачанного.
3. Повторяем 10 раз, суммируя байты и время. Несколько прогонов нужны, чтобы
   сгладить разгон TCP-соединения и джиттер сети.
4. Скорость = суммарные байты / суммарное время, переводим в МБ/с (и в Мбит/с).

К URL добавляется мусорный query-параметр (`?_=<timestamp>`), чтобы CDN/прокси
не отдавал файл из кеша и мы мерили реальную сеть.

## Точность замера

Чем тяжелее файл, тем точнее результат. Общее время запроса состоит из
накладных расходов (DNS, TCP/TLS-рукопожатие, задержка сервера) и времени
передачи самих данных. Накладные расходы примерно постоянны и не зависят от
размера файла. На лёгком файле они доминируют, и по сути меряешь пинг, а не
пропускную способность. На тяжёлом файле время передачи перевешивает, и цифра
приближается к реальной скорости канала. Для адекватного замера нужен файл хотя
бы на несколько мегабайт.

## Запуск

1. Клонируем репозиторий
```
git clone https://github.com/minorytanaka/speedtest.git
```

2. Переходим в каталог
```
cd speedtest
```

3. Создаём и активируем виртуальное окружение
```
# Git Bash (Windows)
python -m venv venv
source venv/Scripts/activate

# Linux / macOS
python3 -m venv venv
source venv/bin/activate
```

4. Ставим зависимости
```
pip install -r requirements.txt
```

5. Запускаем
```
python main.py <url>

python main.py https://i.pinimg.com/736x/d6/c1/26/d6c126c73600d4a81caaddbbcb6086f6.jpg
```
